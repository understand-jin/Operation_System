# 📊 재고 요약표 합산 방식 가이드 (수직 vs 수평)

요약표의 숫자들이 어떤 방향으로 계산되어 만들어지는지 알기 쉽게 정리해 드립니다.

---

## 1. 수직(Vertical) 합산: "위아래로 다 더하기"
**원본 데이터의 뿌리**를 찾는 방식입니다. 주로 표의 **왼쪽 지표들**과 **각 분기별 기본 수치**를 만들 때 사용됩니다.

*   **방식**: 시뮬레이션 원본 전체 리스트에서 해당되는 항목들을 골라 위아래로 덧셈합니다.
*   **해당 지표**:
    *   **원가 / 출하원가 / 출하판가**: 각 자재의 금액을 전체 데이터에서 수직으로 합산.
    *   **각 분기 데이터 (예: 26년 1Q)**: 해당 분기에 배정된 모든 자재의 수량을 수직으로 합산.
*   **특징**: 표의 '재료'가 되는 가장 기초적인 합계입니다.

---

## 2. 수평(Horizontal) 합산: "옆으로 다 더하기"
표에 이미 차려진 **결과값들을 요약**하는 방식입니다. 표의 **가장 오른쪽 [합계] 컬럼**을 만들 때 사용됩니다.

*   **방식**: 표의 한 줄(행)에서 가로로 나열된 분기별 숫자들만 쭉 더합니다.
*   **해당 지표**:
    *   **[합계] 컬럼**: (26년 1Q + 26년 2Q ... + 28년 4Q) = 가로 합계.
*   **특징**: 
    *   시뮬레이션 범위(화면) 안에 들어온 숫자들만 더하기 때문에 **"눈에 보이는 것들의 합"**이 됩니다.
    *   총계 행도 이제 이 방식을 사용하므로, 화면에 보이는 분기별 칸들을 다 더하면 옆의 합계와 정확히 일치합니다.

---

## 💡 한눈에 보는 비교표

| 구분 | 수직(Vertical) 합산 | 수평(Horizontal) 합산 |
| :--- | :--- | :--- |
| **계산 방향** | 위 → 아래 (컬럼 전체) | 왼쪽 → 오른쪽 (행 내부) |
| **적용 영역** | 원가, 출하원가, 분기별 기초 수치 | **오른쪽 끝 [합계] 컬럼** |
| **계산 목적** | 해당 카테고리의 **전체 규모** 파악 | 시뮬레이션 기간 내의 **누적치** 파악 |
| **비유** | 전교생의 몸무게를 다 더하기 | 철수의 1~4분기 몸무게 변화를 더하기 |

---

## 🧐 왜 이 차이가 중요한가요?
1.  **시야의 차이**: 수직 합산은 데이터의 "전체"를 보지만, 수평 합산은 우리가 설정한 "표의 범위(시뮬레이션 기간)" 안만 봅니다.
2.  **데이터 일관성**: 이제는 총계 행도 **수평 합산**을 하도록 수정했기 때문에, 표에 나타난 분기별 숫자들을 수동으로 더해도 옆의 합계와 똑같이 나오게 됩니다. (과거 범위 밖 데이터는 합계에서 제외됨)
